FastHenry2 Automation Overview

FastHenry2 provides a Windows Automation interface which allows a remote application to fully control FastHenry2.

For example, a Visual Basic script inside an MS Windows Excel sheet can launch FastHenry2 in background, make it perform some simulation, then retrieve the results and show them in some cells - and all without even showing up the FastHenry2 window!

The Automation Object

FastHenry2 can be controlled by any Automation-capable application. The automation object which must be created to control FastHenry2 is named FastHenry2.Document. This name is defined in the Windows Registry at installation time or, if removed from the Registry, any time FastHenry2 is run.
